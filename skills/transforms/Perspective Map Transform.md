# Perspective Map Transform

## Description

These instructions describe the configuration and usage of the Perspective Map Transform, a binding tool designed to convert an incoming data value into a different output. This guide explains how to build a mapping table that evaluates the input against a prioritized list of conditions—such as an exact value, a numeric range, or an expression—to return a corresponding new value, color, or component style. This process allows for the dynamic and conditional modification of component properties based on underlying data.

## Documentation

# Instructions
### Perspective Map Transform

The Map Transform is a powerful tool used within bindings to convert an incoming value into a different output value. It works by creating a mapping table where you define specific inputs (or ranges of inputs) and what they should be converted to. This is extremely useful for dynamically changing component properties, like color or text, based on a data value (e.g., a Tag value).

For example, you can map a machine's state integer (0, 1, 2) to display text ("Stopped", "Running", "Faulted") or to a specific color to indicate its status visually.

### Core Concepts

**Mapping Table:** The heart of the Map Transform is a list of mappings. Each mapping defines a relationship between an input and an output.

**Execution Order:** The mappings are evaluated in order from top to bottom. The transform will use the output from the *very first* mapping that matches the incoming value. Any subsequent matches are ignored.

**Fallback Value:** A `fallback` value is a default output that will be used if the incoming value does not match any of the inputs defined in your mapping table. It is critical to set a fallback to gracefully handle unexpected or invalid input values.

**Input Types (`inputType`)**

You can configure the transform to evaluate the incoming value in several ways:

*   **`primitive` (Value):** This is used for an exact match. The incoming value must be identical to the `input` value specified in the mapping. This works for numbers, strings, and booleans.
*   **`range` (Numeric Range):** This checks if an incoming numeric value falls within a specified range. A range is defined by a `min` and `max` value.
    *   You can control whether the `min` and `max` are inclusive or exclusive using the `exclusiveMin` and `exclusiveMax` boolean properties. For example, `(0, 25]` in the documentation translates to: `min: 0, max: 25, exclusiveMin: true, exclusiveMax: false`.
    *   Omitting a `min` or `max` value creates an open-ended range. For example, a range of `(,25]` checks for any value less than or equal to 25.
*   **`expression`:** This allows you to use Ignition's expression language to evaluate the incoming value. The input for the mapping is an expression that returns `true` or `false`. If the expression returns `true`, the mapping is a match. The original incoming value is available in the expression via the `value` variable.

**Output Types (`outputType`)**

Once a match is found, the transform provides an output. The type of output can be one of the following:

*   **`primitive` (Value):** A static, simple value like a string, a number, or a boolean.
*   **`color`:** A color value, represented as a hex string (e.g., "#FF0000" for red).
*   **`inline-style` (Style):** A JSON object that defines a set of style properties to be applied directly to the component. This is useful for applying simple, one-off style modifications without creating a full Style Class.
*   **`style-list` (Style Class):** A string (or array of strings) specifying the name of one or more pre-defined Style Classes to apply. This is the preferred method for applying complex or reusable styles.
*   **`expression`:** An Ignition expression that is evaluated to generate the final output value.

### Configuration Properties

A Map Transform is an object with the following properties:

*   **`inputType`**: A string specifying how to evaluate inputs. Must be one of `primitive`, `range`, or `expression`.
*   **`outputType`**: A string specifying the type of output. Must be one of `primitive`, `color`, `inline-style`, `style-list`, or `expression`.
*   **`fallback`**: A value to be used if no mapping matches. The type of this value should match the selected `outputType`.
*   **`mappings`**: An array of mapping objects. Each object in the array contains an `input` and an `output` property.
    *   **`input`**: The value, range, or expression to check against the incoming value.
    *   **`output`**: The value, color, style, or expression to return if the input matches.

### Helpful Tips & Best Practices

*   **First Match Wins:** Remember that the mappings are executed from top to bottom. The first match found is the one that is used. The order of your mappings is critical.
*   **Always Use a Fallback:** It is best practice to always configure a `fallback` value. This prevents errors and ensures your component has a predictable state even when the binding returns an unexpected value.
*   **Inclusive vs. Exclusive Ranges:** When using the `range` input type, pay close attention to the `exclusiveMin` and `exclusiveMax` properties to ensure your ranges behave as expected. `exclusiveMin: true` is equivalent to `(` and `exclusiveMin: false` is equivalent to `[`.
*   **Style Class vs. Inline Style:**
    *   For complex or reusable styles, it is better to define a Style Class in the Perspective Styles section and use the `style-list` output type to apply it.
    *   Use the `inline-style` output type for simple, unique style changes that won't be reused elsewhere.
*   **Chaining Transforms:** The output of a Map Transform can be passed as the input to another transform (like a Format or Script transform) on the same binding, allowing for complex data manipulation.
*   **Expressions for Complex Logic:** The `expression` input type is very powerful. It allows you to use any of Ignition's built-in expression functions to create complex conditions for your mappings.
