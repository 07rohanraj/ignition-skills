# Perspective XY Chart Component

## Description

These instructions detail the configuration of the Perspective XY Chart component, explaining how its core properties—data sources, axes, and series—are used to create a wide variety of visualizations from basic line charts to complex Gantt, candlestick, and heatmap displays.

## Documentation

# Instructions
# Perspective XY Chart Component

The XY Chart component is a highly versatile tool used to display data trends in Ignition Perspective. It can be configured to show various chart types, including time-series data, category-based data, and more complex visualizations like candlestick charts, Gantt charts, heatmaps, and line charts with target zones.

## Core Concepts

The XY Chart is configured through several key properties that work together to define its appearance and behavior. Understanding these properties is crucial for effective use.

1.  **`dataSources`**: This is an object that holds all the data for the chart. You can have multiple data sources, each identified by a unique key (e.g., `example`, `schedule_data`). The value associated with each key is typically an array of objects, where each object represents a data point. For time-based charts, it is highly recommended that the data is pre-sorted in ascending order by timestamp.

2.  **`xAxes` and `yAxes`**: These are arrays that define the chart's axes. You can have multiple X and Y axes. Each axis configuration is an object within the array and MUST have a unique `name`. This `name` is used to link a series to its corresponding axes. The most important property within an axis is `render`, which can be set to:
    *   `date`: For displaying time-series data. Requires a valid date format.
    *   `value`: For displaying numerical data on a linear or logarithmic scale.
    *   `category`: For grouping data into distinct categories and allotting equal space for each.

3.  **`series`**: This is an array of objects where each object defines how a set of data is plotted on the chart. A series connects a `dataSource` to specific axes and determines the visual representation of the data. Key properties for each series object include:
    *   `data.source`: The key name of the data source from the `dataSources` object.
    *   `data.x` and `data.y`: The keys within your data objects that map to the X and Y values.
    *   `xAxis` and `yAxis`: The `name` of the X and Y axis configurations to use for this series.
    *   `render`: The most critical property in a series. It defines the chart type for that series and dynamically makes other properties available for configuration. Common render types are `line`, `step line`, `column`, and `candlestick`.

## Configuration Examples

### Basic Line Chart

1.  **Data:** Add your data to `dataSources`. For example, create a source named `my_data` with objects containing `t_stamp` and `process_temp` keys.
2.  **X-Axis:** In `xAxes`, create an axis.
    *   Set `name` to "timeAxis".
    *   Set `render` to "date".
    *   Set `date.inputFormat` to match the format of your `t_stamp` data.
3.  **Y-Axis:** In `yAxes`, create an axis.
    *   Set `name` to "valueAxis".
    *   Set `render` to "value".
4.  **Series:** In `series`, create a series object.
    *   Set `data.source` to "my_data".
    *   Set `data.x` to "t_stamp" and `data.y` to "process_temp".
    *   Set `xAxis` to "timeAxis" and `yAxis` to "valueAxis".
    *   Set `render` to "line".

### Candlestick Chart

1. Objective: To create a valid JSON configuration for a Perspective XY Chart component that correctly renders a candlestick chart.

2. Core Components: A valid configuration requires four main top-level properties: dataSources: Holds the raw data. xAxes: Defines the horizontal axis. yAxes: Defines the vertical axis. series: Defines how the data is visualized.

3. Key Configuration Steps:

    dataSources:
        Define at least one data source with a unique name (e.g., "stock_data").
        The data must be an array of objects.
        Each object must contain fields for the date/time, open, high, low, and close values.

    xAxes and yAxes:
        Define at least one x-axis and one y-axis, each as an object within their respective arrays.
        Each axis must have a unique name property (e.g., "dateAxis", "priceAxis"). This name is used to link the series to the axis.
        Set the render property appropriately ("date" for the time-based axis, "value" for the price axis).

    series (The Most Critical Part):
        This is an array that defines each data series on the chart. For a candlestick chart, there will typically be one series object.
        Set render to "candlestick". This is the primary trigger for the candlestick visualization.

        Set the xAxis and yAxis properties to the name of the axes you defined earlier.
        Crucially, add a candlestick object. This object tells the chart which data fields to use for the other parts of the candle:
            It must contain open, high, and low objects.
            Each of these objects must map its y property to the corresponding data field name. For example: "open": { "y": "open" }.

### Gantt Chart

A Gantt chart can be created by using the `column` render type in a specific way.

1.  **Data:** Your data objects need a key for the category/lane (e.g., `name`), a start time (`fromDate`), and an end time (`toDate`). You can also include a `color` key to dynamically color the bars.
2.  **Axes Configuration:**
    *   **Y-Axis:** Set `render` to `"category"` to create the lanes for the Gantt chart. Give it a name (e.g., "Operator").
    *   **X-Axis:** Set `render` to `"date"` to represent the timeline. Give it a name (e.g., "time").
3.  **Series Configuration:**
    *   Set `render` to `"column"`.
    *   **Critical Mapping:**
        *   `data.y`: Map to the category key (`name`).
        *   `data.x`: Map to the **end time** key (`toDate`).
        *   `column.open.x`: Map to the **start time** key (`fromDate`). This is what creates the horizontal bar.
    *   **Dynamic Coloring (Optional):** To use the `color` key from your data, set `column.appearance.deriveFieldsFromData.fill.color` to `"color"`.

### Heatmap

A heatmap is created using a `column` series with `category` axes and `heatRules`.

1.  **Data:** Your data objects need a key for the X-axis category (`year`), a key for the Y-axis category (`fruit`), and a numerical value key to determine the color (`count`).
2.  **Axes Configuration:**
    *   **X-Axis:** Set `render` to `"category"`.
    *   **Y-Axis:** Set `render` to `"category"`.
3.  **Series Configuration:**
    *   Set `render` to `"column"`.
    *   Navigate to `column.appearance.heatRules` and set `enabled` to `true`.
    *   Set `heatRules.dataField` to the numerical value key from your data (`count`).
    *   Set `heatRules.min` and `heatRules.max` to the colors you want for the start and end of the color gradient.

### Line Chart with Target Area

You can create a shaded "target zone" to represent an acceptable range for your data. This is achieved by layering a transparent, filled line series on top of your main data series.

1.  **Data:** Your data objects need a timestamp (`date`), the actual value (`actual`), a high limit (`highLimit`), and a low limit (`lowLimit`).
2.  **Base Series:** Configure a standard line series to plot your `actual` value against the `date`.
3.  **Target Zone Series:** Add a *new* series object to the `series` array.
    *   Set `data.source` to your data source.
    *   Set `xAxis` and `yAxis` to the same axes as your base series.
    *   Set `render` to `"line"`.
    *   Map `data.x` to your date key (`date`).
    *   Map `data.y` to your **high limit** key (`highLimit`).
    *   Map `line.open.y` to your **low limit** key (`lowLimit`). This tells the chart to create a fill between the high and low limit values for each data point.
    *   Set `line.appearance.fill.color` to a desired color (e.g., `#FF8C00`).
    *   Set `line.appearance.fill.opacity` to a low value (e.g., `0.125`) to make the fill semi-transparent.
    *   To hide the lines for the target zone, you can set `line.appearance.stroke.width` to `0`.

## Helpful Tips

*   **Linking Properties by Name:** The `name` property is essential for connecting different parts of the chart. A series uses the `name` of an axis to bind to it (`series.xAxis`, `series.yAxis`). Cursors (`cursor.series`) and scrollbars (`scrollBars.horizontal.series`) also use the `name` of a series to link to it. Ensure these names are unique and correct.
*   **Dynamic `render` Property:** Remember that setting the `series.render` property (`line`, `column`, etc.) dynamically changes the available properties within that series object in the designer. You must set `render` *first* before you can configure type-specific properties like `column.appearance` or `candlestick.open`.
*   **Data-Driven Appearance:** Use the `deriveFieldsFromData` property (e.g., `series.column.appearance.deriveFieldsFromData`) to control visual aspects like fill color, stroke color, or opacity directly from fields in your `dataSources`. This is a powerful way to create dynamic visualizations.
*   **Custom Tooltips:** The `tooltip.text` property on axes and series can use formatting strings to display rich information. You can reference any key from the corresponding data object, for example: `"{name}: [bold]{fromDate} - {toDate}[/]"`.
*   **Multiple Axes and Series:** Don't hesitate to use multiple axes and series on a single chart. You can plot different data sets with different scales. Use the `appearance.opposite: true` property on an axis to move it to the other side of the chart.
*   **Data Selection:** The `selection.enabled` property allows users to click on and select data points. Note that this is only supported for series with `render` set to `column`, `candlestick`, or `line` (only if bullets are enabled on the line). The selected data can be read from the `selection.data` property.
*   **Date/Time Formatting:** When using date axes, ensure the `date.inputFormat` property matches the string format of the date/time values in your data source. The `date.format` property controls how the date is displayed on the axis labels.
