# Perspective Time Series Chart Component

## Description

The usage and configuration of the Perspective Time Series Chart component for visualizing time-series data. These instructions detail the component's hierarchical structure, explaining how to link data `series` to visual `trends` within one or more `plots` and their corresponding `axes`. It also covers the customization of chart types, interactive features like zooming and the x-trace display, and the addition of analytical elements such as calculated baselines and static markers.

# Instructions
Here are the instructions for the LLM on how to use the Perspective Time Series Chart Component.

### Object Name
Perspective Time Series Chart Component

### High-Level Description
The Perspective Time Series Chart component is a powerful tool for visualizing time-series data from various sources. It is fundamentally designed with a time-based X-axis and a value-based Y-axis. The component is highly customizable and includes features such as interactive zooming and panning, an "x-trace" hover display for inspecting data points, multiple chart display types (Area, Bar, Line, Scatter), and the ability to configure multiple plots and Y-axes. It can also display baselines calculated from the data (e.g., average, min, max) and static markers. For optimal use, it can be paired with the Chart Range Selector component.

### Key Concepts

The Time Series Chart's structure is composed of a few core concepts that work together:

1.  **`series`**: This is the foundational data layer. The `series` property is an array that holds one or more data sources for the chart. Each object in the array requires a unique `name` and a `data` payload. The data itself can be formatted in several ways:
    *   **As a dataset**: For example, from a Tag History or Named Query binding. In this format, the **first column must be the timestamp**. Subsequent columns are data values.
    *   **As an array of objects**: Each object must have a `time` key for the timestamp, and other keys for the named values (e.g., `{"time": 1560469431423, "Temperature": 52, "Pressure": 12}`).
    *   **As an array of arrays**: Each sub-array represents a row of data, where the **first element must be the timestamp** (e.g., `[1560469432423, 18, 42]`).

2.  **`plots`**: A plot is a horizontal section or "subplot" within the chart that contains trends and axes. A chart can have multiple plots, which are displayed vertically. The `relativeWeight` property of each plot determines its height relative to other plots. Each plot contains its own `trends`, `axes`, and `markers`.

3.  **`trends`**: A trend defines the visual representation of a data series. It is configured within a `plot`. Key properties of a trend include:
    *   `type`: The chart type to render (`line`, `area`, `bar`, or `scatter`).
    *   `series`: The `name` of the data series (from the top-level `series` array) to visualize.
    *   `axis`: The `name` of the Y-axis (defined in the parent `plot`) to plot the data against.
    A single plot can contain multiple trends, which will be layered on top of each other.

4.  **`axes`**: These are the Y-axes against which trend data is plotted. Axes are defined within a `plot`. Each axis needs a `name` so that trends can reference it. You can control its `alignment` (left or right), `min`/`max` range (or have it auto-range), labels, and tick formatting. The X-axis (time axis) is configured globally under the `timeAxis` property.

5.  **Data Flow and Linkage**: The connection between these concepts is critical. The flow is as follows:
    *   Data is provided to the top-level `series` array. Each series is given a unique `name`.
    *   Within a `plot`, a `trend` is created.
    *   The `trend.series` property is set to the `name` of a data series to link the visual trend to its data.
    *   Within the same `plot`, an `axis` is created and given a `name`.
    *   The `trend.axis` property is set to the `name` of that axis to link the trend to a specific Y-axis.
    *   **Crucially, if you make any custom configurations to a `trend`, you must also define the `axis` it will use within the same `plot`**.

### Property Instructions

#### `series` (Array)
This is the primary data source for the chart. It's an array of objects, where each object represents a set of time-series data.
*   `series[n].name`: A string that uniquely identifies the data series. This name is used by trends to reference this data.
*   `series[n].data`: The data payload. Can be a dataset, an array of objects (with `time` key), or an array of arrays (timestamp first).

#### `plots` (Array)
An array of plot objects. The chart must have at least one plot to display anything.
*   `plots[n].relativeWeight`: A number that controls the height of this plot relative to others. Defaults to `1`.
*   `plots[n].trends`: An array of trend objects to display in this plot.
    *   `trends[n].type`: The visual chart type. `enum`: `"line"`, `"area"`, `"bar"`, `"scatter"`.
    *   `trends[n].series`: **Required.** The `name` of the data series to display. Must match a `name` in the top-level `series` array.
    *   `trends[n].axis`: The `name` of the axis (from this plot's `axes` array) to use for the Y-axis. If left blank, an axis will be auto-generated.
    *   `trends[n].columns`: An array used to specify which columns from the data series to display and to apply specific styling overrides. If empty, all columns are shown.
    *   `trends[n].interpolation`: (For `line`/`area` types) The curve drawing style. `enum`: `"curveLinear"`, `"curveStep"`, `"curveMonotoneX"`, etc. Defaults to `"curveLinear"`.
    *   `trends[n].breakLine`: (For `line` type) If true, creates gaps in the line where data is missing. Defaults to `true`.
    *   `trends[n].stack`: (For `area`/`bar` types) If true, multiple data columns in the trend will be stacked. Defaults to `true`.
    *   `trends[n].radius`: (For `scatter` type) The radius of the plotted points. Defaults to `2`.
    *   `trends[n].baselines`: An array of calculated lines (e.g., average) based on the trend's data.
        *   `function`: The calculation to perform. `enum`: `"min"`, `"max"`, `"avg"`, `"ucl"` (Upper Control Limit), `"lcl"` (Lower Control Limit).
        *   `column`: The data column to calculate against.
        *   `axis`: The axis to draw the baseline against.
*   `plots[n].axes`: An array of Y-axis configurations for this plot.
    *   `axes[n].name`: **Required.** A unique string name for the axis. This is referenced by `trends.axis`.
    *   `axes[n].alignment`: The side of the plot to draw the axis. `enum`: `"left"`, `"right"`. Defaults to `"left"`.
    *   `axes[n].min`/`axes[n].max`: The minimum and maximum range for the axis. If not set, it will auto-range based on the data.
    *   `axes[n].width`: The pixel width of the axis. Defaults to `60`.
    *   `axes[n].label`: An object to configure the axis title (text, visibility, font, etc.).
    *   `axes[n].tick`: An object to configure the axis ticks (count, color, label formatting, etc.).
    *   `axes[n].grid`: An object to configure gridlines associated with this axis.
*   `plots[n].markers`: An array of static visual indicators (currently, only lines are supported).
    *   `markers[n].type`: The marker type. `enum`: `"line"`.
    *   `markers[n].axis`: **Required.** The `name` of the axis to draw the marker on.
    *   `markers[n].value`: The numeric value on the specified axis where the marker line should be drawn.
    *   `markers[n].line`: An object to configure the line's appearance (color, width, dash style, label, etc.).

#### `timeAxis` (Object)
Configuration for the shared X-axis at the bottom of the chart.
*   `visible`: Toggles the visibility of the time axis.
*   `height`: The pixel height of the axis area.
*   `tickCount`: The number of ticks to display. Set to `0` or `"auto"` to have it scale automatically.
*   `tick`: An object for configuring the time axis ticks.
    *   `label.format`: A MomentJS string to format the date/time labels (e.g., `"YYYY-M-D h:mm:ss"`). Defaults to `"Auto"`.

#### `xTrace` (Object)
Configuration for the hover-over display that tracks the cursor.
*   `visible`: Toggles the entire x-trace feature.
*   `infoBox`: An object for configuring the data box that appears.
    *   `visible`: Toggles the info box visibility.
    *   `showTime`: Toggles the time value displayed above the box.
    *   `dataFormat`: A Numeral.js string for formatting numeric values in the box (e.g., `"0,0.##"`).
    *   `dateFormat`/`timeFormat`: MomentJS strings for formatting the date/time display in the box.
*   `line`: An object for configuring the vertical tracking line.

#### `legend` (Object)
Configuration for the chart legend.
*   `visible`: Toggles the legend visibility. Defaults to `false`.
*   `position`: The location of the legend. `enum`: `"top"`, `"right"`, `"bottom"`, `"left"`.

#### `title` (Object)
Configuration for the chart's main title.
*   `visible`: Toggles the title visibility. Defaults to `false`.
*   `text`: The title text to display.

#### `enablePanZoom` (Boolean)
If `true`, allows the user to pan by dragging and zoom with the mouse wheel. Defaults to `true`.

### Step-by-Step Configuration Example

1.  **Add Data**: Drag a Time Series Chart component onto a view. The most common way to add data is to bind the `series[0].data` property to a Tag History or Named Query binding.
    *   When using a binding, give your data columns clear aliases (e.g., "temp", "pressure"). These aliases will be the data keys.
    *   Ensure the binding returns a dataset where the first column is a timestamp.
2.  **Name the Series**: Set `series[0].name` to a descriptive name, like "BoilerData".
3.  **Configure the Trend**: Navigate to `plots[0].trends[0]`.
    *   Set `series` to "BoilerData" (the name from step 2).
    *   Choose a `type`, for example, "line".
4.  **Configure the Axis**: By default, an axis is auto-generated. To customize it, navigate to `plots[0].axes[0]`.
    *   Set the `name` property to something descriptive, like "tempAxis".
    *   Set `label.text` to "Temperature".
    *   Set the `alignment` to "left".
5.  **Link Trend to Axis**: Go back to `plots[0].trends[0]` and set the `axis` property to "tempAxis" (the axis name from step 4).
6.  **Add a Second Trend (Optional)**: To plot "pressure" on a separate axis in the same plot:
    *   Add a new axis to `plots[0].axes` (it will be at index 1). Name it "pressureAxis", set its `label.text` to "Pressure", and its `alignment` to "right".
    *   Add a new trend to `plots[0].trends` (it will be at index 1).
    *   Set its `series` to "BoilerData".
    *   Set its `axis` to "pressureAxis".
    *   The `columns` property of each trend can be used to specify *which* column that trend should display (e.g., `plots[0].trends[0].columns[0].key` = "temp", and `plots[0].trends[1].columns[0].key` = "pressure"). If `columns` is left empty, the trend will try to render all numeric columns from its data source.
7.  **Customize Appearance**:
    *   Set `title.visible` to `true` and provide `title.text`.
    *   Set `legend.visible` to `true` to help identify the trends.

### Helpful Tips & Common Pitfalls

*   **The Linkage Chain is Critical**: The most common error is a mismatch between names. Double-check that `plots.trends.series` exactly matches a `series.name` and that `plots.trends.axis` exactly matches a `plots.axes.name` within the same plot.
*   **Data Format**: Ensure your timestamp is the first column in any dataset or array-of-arrays data source. For object-based data, ensure the timestamp key is named `time`.
*   **Customization Requires Explicitness**: While the chart can auto-generate a basic display from the `series` data, as soon as you want to customize a trend (e.g., change its color or type), you **must** explicitly define the axis for it in the `plots.axes` array and link to it.
*   **Markers vs. Baselines**: Remember the distinction:
    *   `baselines` are dynamic and calculated from the data *within a trend* (e.g., average of the temperature data).
    *   `markers` are static and are drawn at a fixed value *on a plot's axis* (e.g., a line at 90 degrees to mark a limit).
*   **Contextual Properties**: Some properties only appear when a certain `type` of trend is selected. For example, `radius` is only available for `scatter` charts, and `breakLine` is only available for `line` charts.
*   **Resetting Zoom/Pan**: When a user is zoomed or panned in, a reset icon will appear in the upper-right corner of the chart, allowing them to return to the default view.
*   **Styling Hierarchy**: Styles are applied in a cascading manner. The top-level `defaultStyles` property sets the base style for all trends. This can be overridden for specific columns within a trend by configuring the `plots.trends.columns[n].styles` property.
