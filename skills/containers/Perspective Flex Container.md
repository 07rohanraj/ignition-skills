# Perspective Flex Container

## Description

These instructions explain the usage and configuration of the Perspective Flex Container, a tool for creating dynamic and responsive layouts. They detail how to set properties on the container to control the overall flow, alignment, and wrapping of its contents, as well as how to configure individual child components to manage their specific size, flexibility, and alignment within the layout.

## Documentation

# Instructions
Here are the instructions for the **Perspective Flex Container**.

### Description
The Perspective Flex Container is a powerful layout tool used to arrange and distribute components within a view, especially when the size of the components or the screen is dynamic. It works by establishing a main axis and a cross axis and arranging its child components along these axes. The container can expand children to fill available space or shrink them to prevent overflow, making it ideal for creating responsive designs that adapt to different screen sizes. The Flex Container's functionality is based on the CSS Flexible Box Layout (Flexbox) model.

### Container Properties
These properties are configured on the Flex Container itself.

| Property | Type | Description |
| :--- | :--- | :--- |
| **direction** | String | Determines the main axis along which child components are placed. <br> - `row`: (Default) Children are placed horizontally from left to right. <br> - `row-reverse`: Children are placed horizontally from right to left. <br> - `column`: Children are placed vertically from top to bottom. <br> - `column-reverse`: Children are placed vertically from bottom to top. |
| **wrap** | String | Controls whether child components are forced onto a single line or can wrap onto multiple lines if they run out of space along the main axis. <br> - `nowrap`: (Default) All children are forced onto one line. <br> - `wrap`: Children will wrap onto another line if needed. <br> - `wrap-reverse`: Children will wrap onto another line in the reverse order. |
| **justify** | String | Distributes extra space along the **main axis**. This property defines how children are aligned horizontally if `direction` is `row` or vertically if `direction` is `column`. <br> - `flex-start`: (Default) Children are packed toward the start of the `direction`. <br> - `flex-end`: Children are packed toward the end of the `direction`. <br> - `center`: Children are centered along the main axis. <br> - `space-between`: Children are evenly distributed; the first item is on the start line, the last item is on the end line. <br> - `space-around`: Children are evenly distributed with equal space around them. <br> - `space-evenly`: Children are distributed so that the spacing between any two items is equal. |
| **alignItems** | String | Aligns children along the **cross axis**. If `direction` is `row`, this controls vertical alignment. If `direction` is `column`, this controls horizontal alignment. <br> - `stretch`: (Default) Children are stretched to fill the container's cross-axis size. <br> - `flex-start`: Children are aligned to the start of the cross axis. <br> - `flex-end`: Children are aligned to the end of the cross axis. <br> - `center`: Children are centered on the cross axis. <br> - `baseline`: Children are aligned by their text baselines. |
| **alignContent** | String | Aligns the container's lines of content when there is extra space on the **cross axis**. This property only has an effect when `wrap` is set to `wrap` or `wrap-reverse` and there are multiple lines of children. <br> - `stretch`: (Default) Lines stretch to take up the remaining space. <br> - `flex-start`: Lines are packed to the start of the container. <br> - `flex-end`: Lines are packed to the end of the container. <br> - `center`: Lines are packed to the center of the container. <br> - `space-between`: Lines are evenly distributed; the first line is at the start, the last is at the end. <br> - `space-around`: Lines are evenly distributed with equal space around each line. |
| **style** | Style Object | Manages the container's appearance, including background colors, borders, and padding. The `overflow` property is set to `auto` by default. |

### Child Component Position Properties
These properties are configured on each individual component placed *inside* a Flex Container.

| Property | Type | Description |
| :--- | :--- | :--- |
| **basis** | String | The default size of a component before remaining space is distributed. If `direction` is `row`, `basis` defines the base width. If `direction` is `column`, it defines the base height. This can be a pixel value (e.g., "100px") or "auto". <br> - `auto`: (Default) The browser looks at the component's width/height properties to determine its initial size. |
| **grow** | Number | A number that dictates how much a component will grow to fill available space along the main axis, relative to its siblings. <br> - `0`: (Default) The component will not grow. <br> - `1` or greater: The component will grow to fill space. The amount it grows is proportional to the `grow` values of its siblings. |
| **shrink** | Number | A number that dictates how much a component will shrink if there is not enough space along the main axis, relative to its siblings. <br> - `1`: (Default) The component will shrink if necessary. <br> - `0`: The component will not shrink and may overflow the container. <br> - Greater than `1`: The component will shrink more than its siblings with a lower shrink value. |
| **align** | String | Overrides the container's `alignItems` property for a single child component, allowing it to be aligned differently on the cross axis. The possible values are the same as `alignItems` (`auto`, `flex-start`, `flex-end`, `center`, `baseline`, `stretch`). `auto` resets to the container's default. |
| **display**| Boolean | If set to `false`, the child component will be completely removed from the layout and will not be visible. |

---

### Helpful Tips
*   **Main Axis vs. Cross Axis:** The `direction` property is the most important setting. It determines the main axis (`row` or `column`). `justify` applies to the main axis, while `alignItems` and `alignContent` apply to the cross axis.
*   **Fixed vs. Flexible Size:**
    *   To give a component a **fixed size** that does not change, set its `position.grow` and `position.shrink` properties to `0`. You should also set a specific `position.basis` (e.g., "150px").
    *   To make a component **flexible**, set its `position.grow` to a value greater than `0`. A value of `1` is common for components that should share extra space equally.
*   **Proportional Sizing:** The `grow` and `shrink` properties are all about ratios. If sibling components A, B, and C have `grow` values of `1`, `1`, and `2` respectively, component C will receive twice as much of the extra space as A or B. The same logic applies to `shrink`.
*   **Even Scaling:** If you want components to maintain their size ratio as the container resizes, ensure their `grow` values have the same ratio as their `basis` values. For example, if Component A has `basis: "100px"` and Component B has `basis: "200px"`, you should set their `grow` values to `1` and `2` respectively to maintain their 1:2 size ratio as they expand.
*   **`basis` is the Starting Point:** `basis` sets the initial size of a component *before* `grow` or `shrink` are applied. Use `auto` to let the component size itself naturally, or a specific pixel/percentage value to enforce a starting dimension.
*   **When to Use `alignContent`:** The `alignContent` property has no effect unless you have multiple rows or columns of components. This only happens when the container's `wrap` property is set to `wrap` or `wrap-reverse`.
*   **Overriding Alignment:** To align a single, specific child differently from its siblings, use that child's `position.align` property. This is useful for centering one button in a row while the others are aligned to the start.
*   **Hiding a Component:** To conditionally hide a component within the flex layout, bind a property to its `position.display` property. Setting it to `false` removes the component as if it were never there.
