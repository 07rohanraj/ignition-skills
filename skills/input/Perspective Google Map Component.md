# Perspective Google Map Component

## Description

These instructions detail the configuration and use of the Perspective Google Map component, covering the initial setup of the map's appearance, its layered system for displaying geographic data like markers and polygons, and the customization of interactive features such as popups. Furthermore, it explains how to handle user interactions through events and control the map's state programmatically with scripting functions.

## Documentation

# Instructions
Here are the instructions for the Perspective Google Map Component.

### Component Overview

The Perspective Google Map component is a powerful and interactive tool for displaying geographic data in Ignition. It allows for the display of various map features through a layered system, including markers, polygons, circles, and more. The map is highly customizable, supporting various controls, event handling, and scriptable interactions.

**CRITICAL NOTE:** To use this component without a "Development Mode" watermark, a Google Map API Key **must** be provided in the `session.props.googleMapsApiKey` property. This key will be exposed on the client-side, so it is imperative to follow Google's API Security Best Practices, such as restricting the key to specific HTTP referrers.

### Core Concepts

The map is built using a system of **layers**. The primary configuration for what appears on the map is done through the `layers` property. This property is an object that contains arrays for different types of map elements you can add, such as markers, circles, polygons, etc.

A common and important pattern for many layer types is a distinction between **shared properties** and **instance properties**:
*   The main layer object (e.g., `layers.marker`) contains properties that apply to all elements within that layer by default (e.g., `layers.marker.draggable = true` makes all markers draggable).
*   Within the main layer object, there is an array property (e.g., `layers.marker.markers`) that holds the configuration for each individual element (an "instance"). Properties set on an instance will override the shared properties. For example, you could make all markers draggable by default but then set `draggable: false` for one specific marker in the `markers` array.

### Top-Level Properties

These are the main properties for configuring the overall map's appearance and behavior.

| Property | Description |
| :--- | :--- |
| **`init`** | **(Required)** An object that defines the map's initial state when it loads. It includes `center` (an object with `lat` and `lng`) and `zoom` (a number). |
| **`mapType`** | An object to configure the map's type. `id` sets the initial type (e.g., `roadmap`, `satellite`, `hybrid`, `terrain`). You can also configure the map type `controls`, their `controlStyle`, and `controlPosition`. |
| **`restriction`** | Defines a boundary to restrict the map area accessible to users. You can set `bounds` (north, south, east, west) and use `strict: true` to prevent zooming out beyond the bounds. |
| **`gestureHandling`** | Controls how the map handles gestures. Options are `cooperative`, `greedy`, `none`, or `auto`. `cooperative` is often best for pages where users might scroll past the map. |
| **`disableDefaultUI`** | A boolean that, when true, disables all default UI buttons (zoom, fullscreen, etc.). These can be re-enabled individually. |
| **`keyboardShortcuts`** | A boolean to enable or disable map control via the keyboard. |
| **`clickableIcons`** | A boolean that determines if built-in Point of Interest (POI) icons on the map (like parks or shops) are clickable. |
| **`zoom`** | An object to configure zoom controls. You can set the `min` and `max` zoom levels and configure the `controls` and their `controlPosition`. |
| **`fullscreen`** | An object to configure the fullscreen control button, including its visibility (`controls`) and `controlPosition`. |
| **`rotate`** | An object to configure the rotate control button. |
| **`scale`** | An object to configure the scale control. |

### Layers In-Depth

All visual elements on the map are configured within the `props.layers` object. You can have multiple layers of each type.

#### **Marker Layers (`layers.marker`)**
Markers identify a specific location on the map.

*   **Instances:** Individual markers are defined in the `layers.marker[n].markers` array.
*   **Key Instance Properties:**
    *   `position`: **(Required)** An object with `lat` and `lng` to place the marker.
    *   `name`: A unique name or ID used to identify the marker in event handlers.
    *   `icon`: Configures the marker's appearance. See "Icon Configuration" below.
    *   `popup`: Configures a popup window that appears on click. See "Popup and Tooltip Configuration" below.
    *   `tooltip`: Configures a tooltip that appears on hover. See "Popup and Tooltip Configuration" below.
    *   `draggable`: Boolean to allow the user to drag this specific marker.
    *   `animation`: Can be set to `bounce` or `drop` to animate the marker. Set to `none` to stop.
    *   `visible`: Boolean to show or hide the marker.
*   **Marker Clustering:**
    *   To improve performance with many markers, set `layers.marker[n].clusterMarkers: true`.
    *   This groups nearby markers into a single cluster icon.
    *   Configure clustering behavior with the `clusterer` object (e.g., `gridSize`, `minimumClusterSize`, `maxZoom`).
    *   **Note:** Popups and tooltips do not apply to cluster icons themselves.

#### **Polygon Layers (`layers.polygon`)**
Polygons are arbitrary closed shapes on the map.

*   **Instances:** Individual polygons are defined in the `layers.polygon[n].polygons` array.
*   **Key Instance Properties:**
    *   `points`: **(Required)** An array of `lat`/`lng` coordinate arrays that define the shape's path. The path is closed automatically. A polygon can have multiple paths (e.g., to create a shape with a hole in it).
    *   `name`: A unique name for event handling.
    *   `fill`: An object to configure the fill color (`color`), `opacity`, and if it's `enabled`.
    *   `stroke`: An object to configure the border (`color`, `opacity`, `weight`, `position`).
    *   `editable`: If true, users can edit the shape by dragging its vertices.
    *   `draggable`: If true, users can drag the entire shape.
    *   `geodesic`: If true, edges follow the Earth's curvature.

#### **Polyline Layers (`layers.polyline`)**
Polylines are lines that connect a series of coordinates.

*   **Instances:** Individual polylines are defined in the `layers.polyline[n].polylines` array.
*   **Key Instance Properties:**
    *   `points`: **(Required)** An array of `lat`/`lng` objects that define the line's vertices.
    *   `stroke`: Configures the line's appearance (`color`, `weight`, `opacity`).
    *   `icons`: An array to render symbols along the line (e.g., arrows to show direction). You can configure the `icon` symbol, `offset`, and `repeat` properties.
    *   Other key properties include `name`, `editable`, `draggable`, and `geodesic`.

#### **Circle Layers (`layers.circle`)**
Circles are defined by a center point and a radius.

*   **Instances:** Individual circles are defined in the `layers.circle[n].circles` array.
*   **Key Instance Properties:**
    *   `center`: **(Required)** An object with `lat` and `lng` for the circle's center.
    *   `radius`: **(Required)** The circle's radius in meters.
    *   `fill` and `stroke`: Configuration objects for styling.
    *   Other key properties include `name`, `editable`, and `draggable`.

#### **Rectangle Layers (`layers.rectangle`)**
Rectangles are defined by two opposing corner coordinates.

*   **Instances:** Individual rectangles are defined in the `layers.rectangle[n].rectangles` array.
*   **Key Instance Properties:**
    *   `bounds`: **(Required)** An object with two corners (`corner1` and `corner2`), each with `lat` and `lng`.
    *   `fill` and `stroke`: Configuration objects for styling.
    *   Other key properties include `name`, `editable`, and `draggable`.

#### **OverlayView Layers (`layers.overlayView`)**
This powerful layer type allows you to render any Perspective View directly on the map.

*   **Instances:** Individual views are defined in the `layers.overlayView[n].views` array.
*   **Key Instance Properties:**
    *   `path`: **(Required)** The path to the Perspective View you want to embed.
    *   `position` or `bounds`: You must specify either `position` (to place the view at a single `lat`/`lng` point) or `bounds` (to stretch the view between two `lat`/`lng` corners).
    *   `params`: An object of parameters to pass into the embedded view.
    *   `transparentBackground`: Set to true to remove the view's default background.
    *   `mapPaneName`: Controls the rendering layer, which can affect whether the view appears above or below other map elements like markers. `overlayMouseTarget` is often useful.

#### **Other Layer Types**
*   **`kml`**: Loads and displays a KML or KMZ file from a given `url`.
*   **`groundOverlay`**: Binds and scales a single image to fit projected ground coordinates defined by `bounds`.
*   **`traffic`**: When `enabled`, shows current road traffic information.
*   **`transit`**: When `enabled`, shows public transit lines.
*   **`bicycling`**: When `enabled`, highlights bicycle lanes and paths.

### Icon, Popup, and Tooltip Configuration

#### **Icon Configuration (`icon` property on markers)**
*   `type`: Determines the icon style.
    *   `default`: The standard red Google Maps pin.
    *   `image`: Uses a custom image. You must provide a `url`. You can also specify `size`, `scaledSize`, and `anchor` points.
    *   `symbol`: Uses a vector path. You can choose a built-in `path` (like `circle` or `forward_open_arrow`) or provide a `customPath` using SVG path notation. You can style it with `fill` and `stroke` properties.

#### **Popup and Tooltip Configuration**
These objects allow you to display information when a user interacts with a marker.
*   **`popup`**: Appears on click.
*   **`tooltip`**: Appears on hover.
*   **Key Properties:**
    *   `enabled`: Must be `true` for the popup/tooltip to function.
    *   `content`: An object containing the information to display.
        *   `text`: Simple text content.
        *   `view`: To display a full Perspective View. Provide the `path` to the view and any `params` it needs. This overrides the `text` property if configured.
    *   `permanent` (Tooltip only): If true, the tooltip is always visible, not just on hover.
    *   `autoClose` (Popup only): If false, the popup will not close when another popup is opened.

### Events

You can configure scripts to run on various events. The `event` object passed to the script contains contextual information.

*   **Map Events:** `onMapClick`, `onMapDragEnd`, `onMapZoomChanged`, etc. These provide data like the `latLng` of the event or the new map `zoom` level.
*   **Layer Interaction Events:** These are fired when a user interacts with a specific layer element (e.g., a marker or polygon).
    *   Examples: `onMarkerClick`, `onPolygonDragEnd`, `onCircleRadiusChanged`.
    *   The `event` object for these events will almost always include the `name` of the element that was interacted with, along with other relevant data like the `latLng` of the click or the new position after a drag.

### Scripting Functions

These functions are called directly on the component instance (e.g., `self.getSibling("GoogleMap").setZoom(10)`).

*   `setZoom(zoom)`: Sets the map's zoom level.
*   `setCenter(latLng)`: Centers the map at the given coordinates (e.g., `{'lat': 40.7, 'lng': -74.0}`).
*   `panTo(latLng)`: Smoothly pans the map to the given coordinates.
*   `fitBounds(latLngBounds)`: Sets the viewport to contain the given geographical bounds. The `latLngBounds` is a dictionary with `north`, `south`, `east`, and `west` values.
*   `setHeading(heading)`: Sets the compass heading for aerial imagery.
*   `setTilt(tilt)`: Sets the map's tilt angle (0 or 45).

### Helpful Tips
*   **API Key is Essential:** The component is not functional for production use without a `googleMapsApiKey` set in the session properties. Remember to secure it.
*   **Use the `name` Property:** Always give your layer instances (markers, polygons, etc.) a unique `name`. This is the most reliable way to identify which element triggered an event in your scripts.
*   **Use Views for Rich Popups:** For anything more than simple text, use the `popup.content.view` property. Create a dedicated View for your popup's content, which allows for complex layouts, binding, and components.
*   **Understand Shared vs. Instance Properties:** To configure all elements in a layer the same way, set properties on the main layer object (e.g., `layers.marker[0].draggable`). To configure a specific element, set the property inside its instance object in the corresponding array (e.g., `layers.marker[0].markers[0].draggable`).
*   **Performance:** If you have a large number of markers, enable `clusterMarkers` on the marker layer to significantly improve performance. For slightly smaller but still large sets, `optimized: true` can also help.
*   **Editing Shapes:** To allow users to modify shapes like polygons or circles, you must set both `clickable: true` and `editable: true` on the shape's instance or shared properties.
